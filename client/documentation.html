Project 1 Documentation

GET/HEAD Endpoints

Card Search

  • Relative URL: '/getCard'

  • Supported Methods: GET/HEAD

  • Query Params:
    - 'name' (optional) | the name of the card
    - 'cmc' (optional) | the converted mana cost of the card
    - 'color' (optional) | the color of the card
    - 'rarity' (optional) | the rarity of the card
    - 'type' (optional) | the type of the card
    - 'artist' (optional) | the name of the artist

  • Return Value: Returns an array containing all cards from the current set that match the given combination of parameters.

  • Usage: Find cards that match given criteria (NOTE: Text fields are case sensitive).

  • Examples (from Alpha):
    - '/getCard?name=Lightning%20Bolt' (Returns 1 card named Lightning Bolt)
    - '/getCard?type=Artifact&cmc=0' (Returns 6 cards that are Artifacts with Mana Cost 0)
    - '/getCard?artist=Christopher%20Rush&color=R' (Returns 4 cards that are illustrated by Chris Rush and are Red)


Raw Data

  • Relative URL: '/getRawData'

  • Supported Methods: GET/HEAD

  • Return Value: Returns an array containing every card object in the currently selected set.

  • Usage: View raw json data for testing / usage in other applications.


Random Card

  • Relative URL: '/getRandomCard'

  • Supported Methods: GET/HEAD

  • Return Value: Returns an array containing 1 card object chosen at random from the current set.

  • Usage: Find random cards.


Collection Data

  • Relative URL: '/getCollectionData'

  • Supported Methods: GET/HEAD

  • Return Value: Returns an array containing all card objects with a 'collected' field equal to true.

  • Usage: View cards that the user has marked as being collected.


Token Data

  • Relative URL: '/getTokenData'

  • Supported Methods: GET/HEAD

  • Return Value: Returns an array containing all token objects.

  • Usage: View tokens that the user has created.


POST Endpoints

Switch Set

  • Relative URL: '/switchSet'

  • Supported Methods: POST

  • Body Params:
    - 'name' (required) | the code of the corresponding set name

  • Return Value: Returns an apropriate status code with a success/failure message.

  • Usage: Switch the current pool of cards the user can view.

  • Example Set Codes:
    - 'LEA' (Alpha)
    - 'ATQ' (Antiquities)
    - 'FEM' (Fallen Empires)


Add/Remove Card

  • Relative URL: '/addCardToCollection'

  • Supported Methods: POST

  • Body Params:
    - 'name' (required) | the exact name of the card

  • Return Value: Returns an apropriate status code with a success/failure message.

  • Usage: Flips a cards 'collected' field (i.e. sets it to true if false, or sets to false if true).

  • Examples:
    - 'Lightning Bolt' (Returns 204 Status)
    - 'Lihgtning Blat' (Returns 400 Status)


Create Token

  • Relative URL: '/addTokenToCollection'

  • Supported Methods: POST

  • Body Params:
    - 'name' (required) | the name of the token
    - 'description' (required) | the description and characteristics of the token

  • Return Value: Returns an apropriate status code with a success/failure message.

  • Usage: Allows a user to create custom tokens and track them.

  • Examples:
    - 'Name: Goblin' 'Description: 1/1 Red Creature with Haste' (Returns 201 Status)
    - 'Name: Angel' 'Description:' (Returns 400 Status)
